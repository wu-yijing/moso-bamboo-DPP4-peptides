## A. 关键一致性硬伤（投稿前必改）

### A1. 【最严重】多靶点框架的 DPP4 轴定义自相矛盾
- **现象**：§3.11 表 8 中 DPP4 行的 `corr(score, length) = +0.155`（正相关，越长越高分）；但 §3.9 明确写 SCM 分 ~ 长度 ρ = **−0.188**（负相关，越短越高分），且 §3.11 第 254 行自己也说"与 §3.9 DPP4 基准…SCM 分数与长度**负相关**方向相反"。
- **根因**：多靶点矩阵里的"DPP4 归一分"（表 9：LPPGP 0.438 / APQIP 0.330 / APPSQ 0.242）与 §3.13 的"iDPPIV-SCM 概率 0.490 / 排名 77/4742"**不可能是同一分数**——若 LPPGP 在 SCM 中排名 77/4742（前 1.6%，归一应≈0.98），其"DPP4 归一分"却只有 0.438（43.8 百分位）。说明多靶点 DPP4 轴实际是**与 ACE/αG 同源的"参考集签名法"（cosine + 3-mer Jaccard）**，而非 §2.9 表中所称的"基准验证预测器 iDPPIV-SCM"。
- **修复（推荐方案 A，改动最小且更诚实）**：
  1. §2.9 三轴表中，将 DPP4 轴来源改为"**参考集签名法（与 ACE/αG 同源，非 SCM 预测器）**"，删去"基准验证的预测器（可靠）"标签；
  2. 在 §3.11 明写："多靶点框架的三轴**统一采用参考集签名法**以保证可比性；经独立验证的 iDPPIV-SCM 预测器得分单独在 §3.9/§3.13 报告"；
  3. 表 8 DPP4 `corr=+0.155` 保留（与签名法正相关自洽），但 §3.11 第 254 行"与 §3.9 负相关方向相反"改为"三轴在签名法下**均为正相关**，而 §3.9 的 SCM 预测器本身为**负相关**——两种 DPP4 分数的长度行为不同，正说明预测器与签名法捕获了不同信号"。
  - 此改法反而**强化诚实叙事**：ACE/αG 本无验证预测器，三轴同法最透明；SCM 作为已验证 DPP4 专用分单列。

### A2. §3.7 "iDPPIV 双第一" 与 §3.13 绝对排名冲突
- **现象**：§3.7 表/行 156 称 LPPGP"（MM-GBSA、iDPPIV）**双第一**"；但 §3.13 显示 LPPGP 的 iDPPIV-SCM **排名 77 / 4,742**（前 1.6%，并非"第一"）。
- **修复**：将"双第一"改为"**三决赛肽中 iDPPIV 概率最高**（0.490）"，或"**综合优先级居首**"，避免与绝对排名 77 撞车。

### A3. 候选池数量 4,742 vs 4,717 未解释
- **现象**：§3.1 / §3.13 称 SCM 阳性池 4,742 条；§3.11 多靶点矩阵却分析 **4,717** 条（差 25 条）。
- **修复**：补一句说明（预期为含非标准字符或三轴 3-mer 重叠全为 0 的 25 条被剔除），或在脚本中确认后写明剔除规则。

### A4. APPSQ "命中靶点数 0" 却被称为 "DPP4 侧重"
- **现象**：表 9 中 APPSQ `命中靶点数 = 0`（MULTI 标记 none），但 §3.11 第 271 行称"APPSQ / APQIP 为 **DPP4 侧重**候选"。按框架自身"前 10% 即命中"规则，APPSQ 在 DPP4 签名轴（0.242，三决赛中最低）未进前 10%，故 0 靶点是自洽的，但"DPP4 侧重"措辞与它自己 0 靶点矛盾。
- **修复**：把"DPP4 侧重"限定为"**按 §3.7/§3.9 的 SCM 与对接优先级，APPSQ 居 DPP4 单靶点次席**"，与多靶点签名的 0 靶点明确区分两句说。

### C5. 方法用现在时描述已在他环境完成的对接

- **修复**：加一句"上述 DPP4 对接与 MM-GBSA 计算在独立计算环境完成，全部输入结构、受体 PDBQT、脚本与产物已随稿归档以保证可复现"，避免审稿人质疑"为何方法章写的工具当前不可用"。
